# KMP
KMP 是字符串模式匹配算法，核心思想是利用已经匹配过的信息，避免主串指针回退。复习 KMP 时要重点理解 next 数组或前缀函数的含义：它表示模式串当前位置之前，最长相等真前缀和真后缀的长度。考试中常见问题包括手算 next 数组、分析匹配过程、比较 KMP 与朴素匹配的时间复杂度。KMP 的整体时间复杂度是 O(n+m)，适合处理长文本中的模式串查找。

# 哈希表
哈希表通过哈希函数把关键字映射到数组下标，从而实现接近 O(1) 的查找、插入和删除。学习哈希表要掌握哈希函数设计、冲突处理方法和负载因子。常见冲突处理包括拉链法和开放定址法，开放定址法又包括线性探测、二次探测等。复习时要注意删除操作不能简单清空位置，否则可能破坏后续探测路径。

# 最短路径
最短路径是图算法中的高频知识点。Dijkstra 适用于边权非负的单源最短路径问题，核心是每次选择当前距离最小且未确定的点进行松弛。Floyd 用于多源最短路径，通常使用三层循环枚举中转点、起点和终点。复习时要重点区分 Dijkstra、Floyd 和 Bellman-Ford 的适用条件，并能手工模拟距离数组的更新过程。

# 拓扑排序
拓扑排序用于有向无环图，常见应用包括课程先修关系、任务依赖和编译顺序。典型做法是统计每个点的入度，把入度为 0 的点加入队列，依次删除这些点并更新相邻节点入度。如果最终输出节点数量小于图中节点数，说明图中存在环。复习拓扑排序时要会根据入度变化过程判断是否存在合法拓扑序。

# 排序算法
排序算法需要掌握快速排序、归并排序、堆排序、插入排序和冒泡排序的基本思想。快速排序通过划分把数组分成左右两部分，平均时间复杂度是 O(nlogn)，但最坏情况可能退化为 O(n^2)。归并排序使用分治思想，稳定但需要额外空间。堆排序利用堆结构选择最大值或最小值，原地但不稳定。复习时要对比时间复杂度、空间复杂度和稳定性。

# B+树
B+树 是数据库索引中常见的多路平衡查找树。它的非叶子节点主要保存索引信息，所有关键字都会在叶子节点出现，叶子节点之间通常通过链表相连，因此非常适合范围查询。复习 B+树 时要重点掌握节点分裂、合并、范围查询路径，以及它和 B树 在数据存储位置、叶子节点结构方面的区别。
